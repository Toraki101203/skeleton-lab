## 2026-07-10 セキュリティ総点検と緊急修正

### やったこと
月次のセキュリティ総点検（Fable 5）で、本番DBに重大な穴が4つ見つかったので塞ぎました。

1. **誰でも「運営事務局」になれた** — 登録画面で「アカウントタイプ」を選べたうえ、DB がその自己申告をそのまま権限にしていた
2. **全患者の予約が匿名で丸見えだった** — 氏名・電話・メール・症状メモが、アプリの公開キーだけで全件取得できた
3. **ログインした誰もが全クリニックを削除・改変できた** — 「ログイン済みなら無条件でOK」というルールが、所有者チェックを上書きして勝っていた
4. **勤怠を誰でも書き換え・削除できた** — 給与計算の根拠が守られていなかった

あわせて、予約のダブルブッキングを DB 側で物理的に禁止する制約も入れました。

### なぜやったか
実際の患者データが入る前に塞ぐ必要があったため。調査時点では利用者4名すべてがテストアカウントで、
悪用された形跡はありませんでした（＝「今漏れている」ではなく「本番稼働した瞬間に漏れる」状態）。

### 何が良くなったか
匿名キーで本番APIを直接叩いて確認した結果、患者情報も会員メールも0件しか返らなくなりました。
一方で予約カレンダーの空き枠表示は今までどおり動きます（個人情報を含まない専用の「窓」を作ったため）。
同時アクセスによる二重予約も、DB が最後の砦として弾くようになりました。
